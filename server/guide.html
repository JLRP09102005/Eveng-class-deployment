Paso 1 · Preparar Git Bash

Cómo instalar Git Bash en Windows para la actividad

En este primer paso solo necesitamos descargar e instalar Git for Windows. Dejamos casi todo por defecto, pero cambiamos el editor a Notepad++ y activamos los enlaces simbólicos.

Descargar Git Bash Ver instrucciones

Configuración recomendada

  • Deja la mayoría de pantallas con la opción predeterminada.
  • Cuando salga el editor, selecciona Notepad++.
  • Activa la opción de symbolic links.
  • Si no tienes Notepad++, instálalo antes de empezar.

Descarga

Usa el instalador de 64 bits para que toda la clase tenga la misma versión.

Archivo: Git-2.54.0-64-bit.exe
Versión: Git for Windows 2.54.0 · 64 bits
Descargar .exe

Instalación paso a paso

Haz doble clic en el instalador y avanza por el asistente siguiendo esta guía.

1

Ejecuta el instalador

Abre el archivo Git-2.54.0-64-bit.exe. Si Windows pregunta permisos, acepta para continuar.

2

Deja casi todo por defecto

En las pantallas de carpeta, componentes, menú de inicio, PATH y terminal, pulsa Next dejando la configuración recomendada.

Importante: no hace falta personalizar nada extraño. Para la actividad nos interesa que todos tengáis una instalación lo más estándar posible.
3

Cambia el editor a Notepad++

Cuando aparezca la pantalla para elegir el editor por defecto, selecciona Notepad++.

Consejo: si Notepad++ no aparece en la lista, instálalo primero y vuelve a lanzar el instalador de Git.
4

Activa los enlaces simbólicos

En la parte final del asistente, marca la casilla para habilitar symbolic links.

5

Termina la instalación

Pulsa Install y luego Finish. Ya tienes Git Bash listo.

Paso 3 — Registrarse en el servidor

Hay que pedirle al servidor que cree tu usuario y carpeta compartida de Samba. El servidor expone un listener HTTP en el puerto 8080.

POST
/create-vm

Crea tu carpeta compartida en red junto a tu usuario y nombre de VM. Este es el endpoint principal.

GET
/status

Muestra qué carpetas y usuarios han sido creados y compartidos por Samba.

GET
/health

Comprueba que el listener HTTP del servidor está activo y respondiendo.

1

Comprueba que el servidor está activo

Git Bash / PowerShell
curl http://IP-Servidor:8080/health

Deberías ver {"status":"ok"}.

2

Crea tu usuario y carpeta compartida

Git Bash
curl -X POST http://IP-Servidor:8080/create-vm \
  -H "Content-Type: application/json" \
  -d '{"folder":"FOLDER-NAME","vmname":"VM-NAME","username":"USERNAME","password":"PASSWORD"}'
Ejemplo real
Ejemplo
curl -X POST http://192.168.0.100:8080/create-vm \
  -H "Content-Type: application/json" \
  -d '{"folder":"JoseLuisRuizPerez","vmname":"eve-jose","username":"jose","password":"MiPassword123"}'
CampoDescripciónEjemplo

Paso 4 — Preparar PowerShell

Por defecto Windows bloquea los scripts externos. Necesitamos cambiar la política de ejecución.

1

Abre PowerShell como administrador

Pulsa Win, escribe PowerShell, haz clic derecho → Ejecutar como administrador.

Importante: debe ser como administrador.
2

Activa la ejecución de scripts

PowerShell (Administrador)
Set-ExecutionPolicy Unrestricted

Escribe Y y pulsa Enter.

3

Verifica el cambio

PowerShell (Administrador)
Get-ExecutionPolicy

La salida debe ser Unrestricted.

Nota de seguridad

En equipos de producción lo recomendable es volver a RemoteSigned al terminar la actividad.

Paso 7 — Crear la VM en Hyper-V

Con la ISO descargada, crearemos la máquina virtual de EVE-NG desde el Administrador de Hyper-V.

1

Abre Hyper-V Manager

2

Nueva máquina virtual

Ve a Acción → Nuevo → Máquina Virtual. En la primera pantalla pulsa Siguiente.

3

Nombre y ubicación

Ejemplo: si registraste eve-jose, la VM se llama eve-jose.
4

Generación

Selecciona Generación 2.

5

6

En el desplegable Conexión selecciona EVE-NG-Switch.

Si no aparece, vuelve al Paso 5 y ejecuta de nuevo setup-client.ps1 como administrador.
7

Disco duro virtual

Ponle el mismo nombre que tu vmname y guárdalo en:

8

Opciones de instalación

Selecciona "Instalar un SO desde una imagen de arranque" y busca la ISO:

Ruta ISO
C:\EVE-NG-Local\eve-ce-6.2.0-4-full.iso
9

Finalizar el asistente

Pulsa Finalizar. No arranques la VM todavía — antes hay que aplicar tres configuraciones obligatorias con PowerShell.

10

Configuración obligatoria — ejecuta en PowerShell como Administrador

Sustituye eve-jose por el nombre exacto de tu VM en los tres comandos:

① Deshabilitar Arranque seguro

Sin este paso la VM no arrancará con la ISO de EVE-NG.

PowerShell — Administrador
Set-VMFirmware -VMName "eve-jose" -EnableSecureBoot Off

② Habilitar virtualización anidada

Imprescindible para que EVE-NG pueda crear sus propios nodos virtuales.

PowerShell — Administrador
Set-VMProcessor -VMName "eve-jose" -ExposeVirtualizationExtensions $true

③ Habilitar MAC Spoofing

Sin esto los nodos de red de EVE-NG no tendrán conectividad.

PowerShell — Administrador
Set-VMNetworkAdapter -VMName "eve-jose" -MacAddressSpoofing On
Una vez ejecutados los tres comandos sin error ya puedes arrancar la VM.
11

Arrancar la VM

Haz doble clic en la VM en Hyper-V Manager, pulsa Iniciar y arrancará el instalador de EVE-NG.

Paso 8 — Subir los cambios al servidor

Al terminar de usar tu máquina virtual, sube el disco .vhdx actualizado a tu carpeta compartida en Samba con sync.ps1 -mode push.

1

Apaga la VM en Hyper-V

Asegúrate de que la VM está completamente apagada. Un disco en uso no se puede copiar.

Importante: usa Apagar desde Hyper-V o desde dentro de la VM — no solo la pausas.
2

Comprueba que el disco está en su carpeta

El .vhdx debe estar en:

Ruta esperada
C:\EVE-NG-Local\tu-vmname.vhdx
Si no está ahí, muévelo antes de continuar.
3

Ejecuta el script en modo push

PowerShell (Administrador)
.\sync.ps1 -mode push

El script verificará que la VM está apagada, montará tu carpeta Samba y copiará el disco al servidor.

4

Espera a que termine

Siguiente sesión — solo pull + usar

Una vez hayas hecho push, en las clases siguientes solo necesitas .\sync.ps1 -mode pull y arrancar la VM. No crearás la VM de nuevo.

Al acabar cada clase — push obligatorio

Si no haces push perderás los cambios de esa sesión porque el disco no habrá sido guardado en el servidor.

Paso 9 — Sesiones posteriores: pull, trabajar y push

Una vez hayas completado la instalación de EVE-NG y subido tu disco virtual al servidor, no vuelves a crear nada desde cero. En cada sesión solo tienes que descargar tu disco, arrancar la VM y subirlo al terminar.

1

Descarga tu disco del servidor (pull)

Al llegar a clase, abre PowerShell como administrador, navega al repositorio y ejecuta:

PowerShell (Administrador)
.\sync.ps1 -mode pull

El script detectará que ya hay un .vhdx en el servidor y lo descargará a C:\EVE-NG-Local\. No volverá a descargar la ISO.

2

Crea la VM apuntando al disco descargado

En el Administrador de Hyper-V, crea una nueva VM exactamente igual que en el Paso 7 — pero en el paso del disco elige "Usar un disco duro virtual existente" y selecciona el .vhdx que acabas de descargar.

Importante: nombre, generación, RAM y switch deben ser idénticos al Paso 7. Solo cambia la opción del disco.
ParámetroValor (igual que siempre)
Nombre de la VMTu vmname registrado
GeneraciónGeneración 2
Memoria RAM8192 MB
Switch de redEVE-NG-Switch
Disco duroUsar disco existente → C:\EVE-NG-Local\tu-vmname.vhdx
3

Arranca la VM y trabaja

Haz doble clic en la VM en Hyper-V Manager y pulsa Iniciar. EVE-NG arrancará con todo tu progreso guardado de la sesión anterior.

4

Al terminar: apaga y sube los cambios (push)

Apaga la VM completamente desde Hyper-V y ejecuta:

PowerShell (Administrador)
.\sync.ps1 -mode push

El script subirá tu .vhdx actualizado al servidor. En la siguiente sesión, al hacer pull, recuperarás exactamente este estado.

¿Por qué crear la VM cada vez?

Hyper-V guarda las VM localmente en el equipo. Al cambiar de ordenador pierdes la configuración de la VM, pero no el disco — ese vive en el servidor. Por eso el flujo es: descargar el disco, crear la VM apuntando a él, trabajar, y subir el disco de vuelta.
