Paso 1 · Preparar Git Bash

Cómo instalar Git Bash en Windows para la actividad

En este primer paso solo necesitamos descargar e instalar Git for Windows. Dejamos casi todo por defecto, pero cambiamos el editor a Notepad++ y activamos los enlaces simbólicos.

Descargar Git Bash Ver instrucciones

Configuración recomendada

  • Deja la mayoría de pantallas con la opción predeterminada.
  • Cuando salga el editor, selecciona Notepad++.
  • Activa la opción de symbolic links.
  • Si no tienes Notepad++, instálalo antes de empezar.

Descarga

Usa el instalador de 64 bits para que toda la clase tenga la misma versión.

Archivo: Git-2.54.0-64-bit.exe
Versión: Git for Windows 2.54.0 · 64 bits
Descargar .exe

Instalación paso a paso

Haz doble clic en el instalador y avanza por el asistente siguiendo esta guía.

1

Ejecuta el instalador

Abre el archivo Git-2.54.0-64-bit.exe. Si Windows pregunta permisos, acepta para continuar.

2

Deja casi todo por defecto

En las pantallas de carpeta, componentes, menú de inicio, PATH y terminal, pulsa Next dejando la configuración recomendada.

Importante: no hace falta personalizar nada extraño. Para la actividad nos interesa que todos tengáis una instalación lo más estándar posible.
3

Cambia el editor a Notepad++

Cuando aparezca la pantalla para elegir el editor por defecto, selecciona Notepad++.

Consejo: si Notepad++ no aparece en la lista, instálalo primero y vuelve a lanzar el instalador de Git.
4

Activa los enlaces simbólicos

En la parte final del asistente, marca la casilla para habilitar symbolic links.

5

Termina la instalación

Pulsa Install y luego Finish. Ya tienes Git Bash listo.

Paso 3 — Registrarse en el servidor

Antes de configurar nada en local, hay que pedirle al servidor que cree tu usuario y carpeta compartida de Samba. El servidor expone un listener HTTP en el puerto 8080 con tres endpoints.

POST
/create-vm

Crea tu carpeta compartida en red junto a tu usuario y nombre de VM. Este es el endpoint principal.

GET
/status

Muestra qué carpetas y usuarios han sido creados y compartidos por Samba.

GET
/health

Comprueba que el listener HTTP del servidor está activo y respondiendo correctamente.

1

Comprueba que el servidor está activo

Sustituye IP-Servidor por la IP que te haya dado el profesor:

Git Bash / PowerShell
curl http://IP-Servidor:8080/health

Deberías ver una respuesta como {"status":"ok"}. Si no responde, avisa al profesor.

2

Crea tu usuario y carpeta compartida

Envía una petición POST al endpoint /create-vm con tus datos.

Git Bash
curl -X POST http://IP-Servidor:8080/create-vm \
  -H "Content-Type: application/json" \
  -d '{"folder":"FOLDER-NAME","vmname":"VM-NAME","username":"USERNAME","password":"PASSWORD"}'
Ejemplo real
Ejemplo
curl -X POST http://192.168.0.100:8080/create-vm \
  -H "Content-Type: application/json" \
  -d '{"folder":"JoseLuisRuizPerez","vmname":"eve-jose","username":"jose","password":"MiPassword123"}'
CampoDescripciónEjemplo
folderNombre de tu carpeta compartida en SambaJoseLuisRuizPerez
vmnameNombre identificativo de tu VMeve-jose
usernameUsuario Samba que se crearájose
passwordContraseña para ese usuarioMiPassword123
Recuerda estos datos. Son exactamente los que pedirá el script setup-client.ps1 en el paso siguiente.
3

Verifica que tu carpeta fue creada

Git Bash / PowerShell
curl http://IP-Servidor:8080/status

Deberías ver tu folder y vmname listados. Si no aparecen, repite el paso anterior.

Paso 4 — Preparar PowerShell

Antes de ejecutar los scripts, necesitamos configurar PowerShell para que permita su ejecución.

1

Abre PowerShell como administrador

Pulsa Win, escribe PowerShell y haz clic derecho → Ejecutar como administrador. Acepta el aviso de UAC.

Importante: debe ser como administrador. Si lo abres como usuario normal dará error de permisos.
2

Activa la ejecución de scripts

PowerShell (Administrador)
Set-ExecutionPolicy Unrestricted

Escribe Y y pulsa Enter para confirmar.

3

Verifica el cambio

PowerShell (Administrador)
Get-ExecutionPolicy

La salida debe ser Unrestricted.

Nota de seguridad

La política Unrestricted es adecuada para este entorno de prácticas. En producción lo recomendable es volver a RemoteSigned al terminar.

Navegaremos hasta la carpeta del repositorio clonado y ejecutaremos setup-client.ps1.

1

Navega hasta la carpeta clonada

cd $env:USERPROFILE\Desktop\Eveng-class-deployment
2

Ejecuta el script

PowerShell (Administrador)
.\setup-client.ps1
3

El script te pedirá 5 datos uno por uno:

1

2
Nombre de tu carpeta SMBel que usaste en curl

3
Nombre de tu VMel que usaste en curl

4

El nombre de usuario con el que tienes acceso al recurso compartido Samba.

5

Al escribirla puede que no aparezcan los caracteres — es normal, PowerShell la oculta por seguridad.

Espera a que el script termine

Paso 6 — Descargar los archivos del servidor

Con todo configurado, sincronizaremos nuestra carpeta del servidor con el equipo local usando sync.ps1 en modo pull.

1

Ejecuta el script en modo pull

PowerShell (Administrador)
.\sync.ps1 -mode pull
2

Qué descarga el script

Primera vez

El script detecta que no hay ningún .vhdx y descarga automáticamente la ISO de EVE-NG. Con ella crearás la VM en Hyper-V.

Sesiones siguientes

Si ya hiciste un push anterior, el script descargará tu .vhdx con todo tu progreso en la VM de EVE-NG.

3

Espera a que termine

La descarga puede tardar varios minutos. Al terminar verás:

Salida esperada (primera vez)
══════════════════════════════════════════════
  Sincronizacion completada.
  Primera sesion: ejecuta 'EVE-NG Importar VM'.
══════════════════════════════════════════════

Paso 7 — Crear la VM en Hyper-V

Con la ISO ya descargada en tu carpeta local, crearemos la máquina virtual de EVE-NG desde el gestor de Hyper-V de Windows. Sigue el asistente con los valores exactos que se indican.

1

Abre Hyper-V Manager

Pulsa Win, escribe Hyper-V y abre la aplicación Administrador de Hyper-V.

2

Nueva máquina virtual

En el menú superior ve a Acción → Nuevo → Máquina Virtual. En la primera pantalla "Antes de comenzar" pulsa Siguiente.

3

Nombre y ubicación

En el campo Nombre escribe exactamente el mismo vmname que registraste en la petición HTTP. Deja la ubicación por defecto.

Ejemplo: si tu vmname fue eve-jose, la VM debe llamarse eve-jose.
4

Especificar generación

Selecciona Generación 2. Es imprescindible para que EVE-NG arranque correctamente con UEFI.

5

Asignar memoria

Introduce 8192 MB como memoria de inicio.

Valor a introducir
8192
6

Configuración de red

En el desplegable Conexión, selecciona el switch EVE-NG-Switch. Este switch fue creado automáticamente por el script setup-client.ps1 que ejecutaste en el Paso 5.

Si EVE-NG-Switch no aparece en la lista, vuelve al Paso 5 y ejecuta de nuevo setup-client.ps1 como administrador.
7

Disco duro virtual

Pon al disco el mismo nombre que tu vmname y guárdalo en la carpeta especial que creó el script sync.ps1 en C:, cuyo nombre coincide con tu folder registrado.

Ruta de ejemplo
C:\EVE-NG-Local
8

Opciones de instalación

Selecciona "Instalar un sistema operativo desde una imagen de arranque" y localiza la ISO de EVE-NG descargada por el script de sincronización:

Ruta de la ISO
C:\EVE-NG-Local\eve-ce-6.2.0-4-full.iso
9

Finalizar y arrancar

Revisa el resumen y pulsa Finalizar. Haz doble clic en la VM en Hyper-V Manager, pulsa Iniciar y arrancará el instalador de EVE-NG.
